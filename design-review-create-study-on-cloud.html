Design Review — Recording
Create Study On Cloud
UX flow review for enabling users to create predefined study structures (Gas, Electric, Water, Universal) directly from PLEXOS Cloud. Final agreed direction: deep-link users from Cloud to the relevant Marketplace Playbook rather than building a duplicate template system inside Cloud.
Recording
Discussion Notes
Objective & Problem Statement
Goal: Let users quickly create a predefined study structure for Gas, Water, Electric, or Universal studies directly from Cloud.
Current gap: Users must create studies in Excel/Desktop and push them to Cloud — Cloud is not the starting point.
Scope: Creating a basic starting point only — not replacing Marketplace or building a full study ecosystem inside Cloud.
Design Options Discussed
Option 1 — Create Study button on Top Navigation
Add a "Create Study" button in the top band; clicking opens templates.
Rejected: top nav is already crowded (study version, tenant name, notifications); creating studies is not a frequent action.
Option 2 — Create Study inside Study Dropdown
Place "Create Study" inside the existing study dropdown panel.
Concern: new users with no studies would not know to click the dropdown to start — poor discoverability.
New Tenant Scenario
A brand-new customer gets a new tenant with zero studies — a different handling is needed.
Proposed solution: if tenant has no studies, show a modal with recommended starting studies and guide toward first study creation.
Two distinct scenarios must be handled separately: new user (needs onboarding guidance, empty-state modal) vs. existing user (quick access via dropdown).
Marketplace Integration
Starter studies are already being built by a separate team and published to Marketplace as Playbooks (Electric, Gas, and future Water/Universal).
Intent: reuse this work by redirecting users directly to the relevant Marketplace Playbook rather than duplicating templates inside Cloud.
Marketplace contains Insights, Reports, and Playbooks — starter studies are packaged Playbooks with Excel templates.
Final Agreed Direction
User flow: Click Create Study → Select type (Gas/Electric/Water/Universal) → Deep-link to relevant Marketplace Playbook → Click "Add to Cloud" → Study created in tenant.
Critical UX rule: do NOT send users to the full Marketplace browser — link directly to the specific Playbook to avoid confusion.
Open Decisions
Final placement of "Create Study" entry point (top UI / dropdown / conditional empty-state modal) — pending broader team discussion.
Need to validate: how many starter templates will exist long-term (4 vs. 10–20+) and impact on UI scalability.
Need to confirm Marketplace deep-link technical feasibility with engineering.
Need a demo of new tenant creation flow to understand the zero-studies state.
UX Insights
Entry Point Strategy
Avoid adding low-frequency actions (like "Create Study") to a crowded top nav.
Contextual entry points (study dropdown, empty-state modal) reduce noise while improving discoverability for the right users.
Empty State Design
New tenants with zero studies require a different experience — onboarding modal, not just a blank list.
Guided first-run flows prevent users from getting stuck before they understand the product.
Avoid Duplication
Building a new template system inside Cloud when Marketplace Playbooks already exist creates maintenance overhead and UX inconsistency.
Deep-linking reuses existing investment and gives users more value (Playbooks include Excel templates, not just study shells).
