Design Review — Recording
Study Landing Page & TAG — Component
Second round of design review for the Study Landing Page and Tagging system. Covers study card states, user presence indicators, tagging flows, sorting, filtering, search, and the activity panel. Key decisions: remove owner-based sorting, simplify tag management, use panel overlays instead of popups.
Recording
Discussion Notes
Study Card — Three States
Private Study: default, hover, favorite, and favorite+post states; shows title, description, tags, user initials, active editors.
Public Study: anyone can access/edit; only current user's initials shown; same card structure as private.
Requested Access Study: shows study info; hover provides "Request Access" option; expandable for more information.
Access status should be clearly surfaced via badge — Public, Private, and Requested Access all need distinct visual treatment.
Public Private Access Requested
User Presence & Collaboration
Earlier design showed two user groups: left (actively editing), right (assigned but not currently editing).
Decision: active/online users are important; assigned-user list is less critical and should be removed from the card.
Need coordination with Madhu for online-user real-time implementation.
Posts / Comments Behavior
Post indicator: shows initials of who posted; multiple users shown as initials + count (e.g., +2).
Hover: tooltip shows message preview and poster identity — concern about very long messages creating oversized tooltips.
Click behavior: clicking post/comment chip opens a right-side overlay panel (not inline card expansion); panel contains study details, all messages, and reply option.
Tagging System — Add Tag
Entry points: study card hover, or Home screen tag section.
Flow: click "Add Tag" → expand panel showing search, available tags, system tags, user-created tags.
System tags are disabled (non-editable); user tags allow add/remove.
Create & Edit Tag Flow
Click "+" Create Tag → full-screen modal; fields: tag name, color. On Create: new tag appears at top.
Edit: hover tag → Edit icon appears; opens modal; changes apply everywhere the tag is used.
Delete: hover tag → Delete icon appears; confirmation dialog ("Do you want to delete tag?") with Cancel / Delete actions.
"+3 more tags" overflow: avoid tooltip; use a popup with chips; allow delete from within that popup.
Sorting
Removed: "Study Owner" and "Created By" sorting — users care about working on studies, not ownership.
Final sort options: Last Viewed, Study Created Date, Study Title (A–Z / Z–A). Same approach for Tenant Studies.
Study Activity Panel
Reuse existing PLEXOS Cloud activity card — avoid introducing new component patterns.
Sort options: Date/time (Newest/Oldest), User (A–Z / Z–A), Study (A–Z / Z–A).
Filters: Study name, Username. Click activity item → opens related study in new tab.
Global Search & Quick Filters
Search: user clicks → cursor appears → enters keyword → suggestions appear (Studies, Owners, Related results). Results refine as user types.
Quick filters (My Studies): All, Favorite, Public, Private, Not Accessed, Unread Post.
Archive filters: remove Favorite, Public/Private; keep only archive-relevant filters.
UX Insights
Panel Overlay vs. Popup
Replacing popups with right-side overlay panels keeps users in context of the selected study — they don't lose their place in the list.
Panels also give more space for detailed content (messages, tags, activity) that popups can't accommodate cleanly.
Simplify Sorting
Removing "Study Owner" and "Created By" from sort options reflects how users actually think — by recency or title, not by who owns it.
Fewer sort options reduce decision fatigue without removing anything users care about.
Presence Over Assignments
Showing who is actively editing now (online presence) is more useful than the full assigned-user list.
Card space is limited — prioritize real-time information over static metadata.
Tag Management Principles
Consistent entry points (card hover and Home screen) ensure discoverability regardless of user workflow.
Global tag edits that propagate everywhere respect data consistency — avoid tags with different names across the same study set.
